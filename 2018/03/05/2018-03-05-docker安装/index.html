docker安装

发表于 2018-03-05

0x00 docker版本

CE 社区版（免费）
EE 企业版，安全，付费

安装环境为ubuntu16.04

tips：16.04+的版本无需设置内核模块

0x01 APT安装

更新apt软件包缓存

安装apt

$ sudo apt-get install \
apt-transport-https \
ca-certificates \
curl \
software-properties-common

添加软件源的 GPG 密钥，为了确认所下载软件包的合法性

然后，我们需要向 source.list 中添加 Docker 软件源

tips：鉴于国内网络问题，强烈建议使用国内源

0x02 安装docker CE

更新 apt 软件包缓存，并安装 docker-ce：

$ sudo apt-get update

$ sudo apt-get install docker-ce

启动 Docker CE

本机为ubuntu16.04

$ sudo systemctl enable docker
$ sudo systemctl start docker

Ubuntu 14.04 使用以下命令启动：

$ sudo service docker start

建立 docker 用户组

默认情况下，docker 命令会使用 Unix socket 与 Docker 引擎通讯。而只有 root 用户和 docker 组的用户才可以访问 Docker 引擎的 Unix socket。出于安全考虑，一般 Linux 系统上不会直接使用 root 用户。因此，更好地做法是将需要使用 docker 的用户加入 docker 用户组。

建立 docker 组：

$ sudo groupadd docker

将当前用户加入 docker 组：

$ sudo usermod -aG docker $USER

退出当前终端并重新登录，进行如下测试。

0x03 遇到的问题

当我使用当前用户启动docker时，显示如下信息

该错误消息告诉您当前用户无法访问docker引擎，因为您缺少访问unix套接字以与引擎通信的权限

原因是我将用户加入用户组后没有重启！！！重启下就好了

如果大家遇到这种报错可以参考以下两种方案

解决方案：

  • 每次使用sudo以root身份运行（不推荐）

  • 将当前用户加入用户组，然后重启，重启！
